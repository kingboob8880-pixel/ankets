# Анкета для пациентов (GitHub Pages)

Статические файлы для **отдельного** репозитория GitHub (не смешивать с репозиторием бэкапов/публикаций планов).

## Текущая версия (schema v2)

Одна анкета из **11 текстовых разделов** (контакты, жалобы, динамика, триггеры, тело, психика, дом, джинн/масс, сглаз, сихр по видам, эмоциональный тон). Настраивается массивом **`surveySections`** в [`config.json`](config.json).

- Экспорт: `exportKind: "rukya_questionnaire_v1"`, **`schemaVersion: 2`**, поле **`sections`** (объекты `s1`…`s11` с полями вопросов), краткое поле **`complaints`**, **`deepseek.userMessage`**.
- Старые JSON со **`schemaVersion: 1`** (чекбокс-анкета и профили лечения) по-прежнему можно импортировать в RUKYA PRO.

## Содержимое

- `index.html` — оболочка
- `config.json` — разделы и подписи полей
- `app.js` — мастер по шагам (11 экранов + финал с JSON)
- `styles.css` — оформление

Текст в `deepseek.userMessage` должен по смыслу совпадать с логикой приложения ([`../js/engine/deepseek.js`](../js/engine/deepseek.js)); при смене структуры проверьте сборку в `buildUserMessage` в `app.js`.

## Публикация

1. Создайте репозиторий под GitHub Pages.
2. Скопируйте в корень (или `docs/`): `index.html`, `config.json`, `app.js`, `styles.css`.
3. **Settings → Pages**: ветка и папка; при `docs/` укажите `pagesRoot` в RUKYA.

## RUKYA PRO

**Настройки → GitHub → Репозиторий анкет** — ссылка на репозиторий со страницей анкеты.

**Импорт JSON** — кнопка **Анкета (Pages)**; вставка файла с экспортом. Для **schema v2** жалобы в карточке собираются из `complaints` + человекочитаемый разбор `sections` ([`../js/ui/import.js`](../js/ui/import.js)).
